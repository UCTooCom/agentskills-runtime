# Agent 错误恢复与自愈系统需求规格

## 项目背景

当前 agentskills-runtime 的 Agent 执行失败后缺乏系统性的恢复机制。对于长时间运行的多 Agent 协作任务，单个 Agent 或步骤的失败不应导致整个任务失败。AI 驱动开发框架需要自愈能力：自动诊断错误、尝试恢复、必要时降级执行，确保任务最终完成。

## 核心问题

1. **无错误分类**: 所有错误同等对待，无法区分可恢复和不可恢复错误
2. **无自动重试策略**: 失败后无智能重试（当前仅同步系统有 RetryManager）
3. **无降级执行**: Agent 失败后无法降级到更简单的方式继续
4. **无错误传播控制**: 子 Agent 错误直接传播到主 Agent，无隔离和转换
5. **无自愈诊断**: 缺乏自动分析错误根因并调整策略的能力
6. **无熔断机制**: 反复失败的 Agent 无熔断保护，持续消耗资源

## 功能需求

### REQ-ERR-001: 错误分类与分级

- 错误分类：
  - `transient`: 暂时性错误（网络超时、资源暂时不可用），可自动重试
  - `recoverable`: 可恢复错误（参数错误、技能缺失），可通过调整恢复
  - `degradable`: 可降级错误（Agent 不可用），可用替代方式完成
  - `fatal`: 致命错误（权限不足、数据损坏），无法恢复
- 错误分级：critical > error > warning > info
- 自动分类规则 + 自定义分类器

### REQ-ERR-002: 智能重试策略

- 扩展 RetryManager 为通用组件（当前仅同步系统使用）
- 支持多种重试策略：
  - 指数退避（当前已实现）
  - 固定间隔
  - 自定义退避函数
  - 抖动（jitter）避免雷群效应
- 重试前自动诊断：分析错误类型，决定是否值得重试
- 重试时可调整参数（如降低并行度、更换模型）
- 最大重试次数可配置，支持按错误类型设置不同策略

### REQ-ERR-003: 降级执行

- Agent 不可用时自动降级：
  - 替换为更简单的 Agent（如复杂 Agent 失败 → 基础 Agent）
  - 替换为直接技能执行（跳过 Agent 封装）
  - 替换为内置工具直接操作
- 技能不可用时自动降级：
  - 使用技能的旧版本
  - 使用功能相似的替代技能
  - 使用内置工具手动完成
- 降级决策记录到日志，便于后续分析

### REQ-ERR-004: 错误传播控制

- 子 Agent 错误不直接传播，经 ErrorGateway 处理：
  - `absorb`: 吸收错误，主 Agent 使用默认值继续
  - `transform`: 转换错误为警告，调整后续计划
  - `escalate`: 升级错误，通知主 Agent 重新决策
  - `isolate`: 隔离错误，标记子任务失败但不影响其他子任务
- 错误传播策略可按 Agent 类型和错误类型配置
- 支持错误聚合：多个同类错误合并为一条

### REQ-ERR-005: 自愈诊断

- 错误发生后自动执行诊断流程：
  1. 分析错误日志和堆栈
  2. 检查相关资源状态（文件、数据库、网络）
  3. 匹配已知错误模式（错误模式库）
  4. 生成诊断报告和修复建议
- 基于诊断结果自动调整：
  - 修改输入参数
  - 更换执行策略
  - 请求用户干预
- 诊断结果存入记忆系统，避免同类错误重复发生

### REQ-ERR-006: 熔断器

- 为每个 Agent 和技能配置熔断器：
  - 关闭状态（正常）：请求正常通过
  - 打开状态（熔断）：请求直接失败，不尝试执行
  - 半开状态（探测）：允许少量请求测试是否恢复
- 熔断触发条件：连续失败 N 次 / 时间窗口内失败率超过阈值
- 熔断恢复条件：半开状态探测成功 / 超时自动进入半开
- 熔断事件通知（WebSocket + 日志）

### REQ-ERR-007: 补偿事务

- 对于产生副作用的操作，记录补偿动作（反向操作）
- 编排失败时按逆序执行补偿动作
- 补偿动作示例：
  - 文件写入 → 删除文件
  - 数据库插入 → 删除记录
  - 技能安装 → 技能卸载
- 补偿事务支持部分成功（部分补偿失败时记录并告警）

## 非功能需求

- 错误分类延迟 ≤ 10ms
- 重试策略执行不阻塞主流程
- 熔断器状态切换延迟 ≤ 5ms
- 诊断流程超时 ≤ 30s
- 补偿事务执行超时 ≤ 60s

## 数据模型

### error_records 表

| 字段 | 类型 | 说明 |
|------|------|------|
| id | bigint | 主键 |
| agent_id | bigint | 发生错误的 Agent |
| task_id | bigint | 关联任务 |
| error_type | varchar(50) | 错误分类 |
| error_level | varchar(20) | 错误级别 |
| error_code | varchar(50) | 错误码 |
| message | text | 错误消息 |
| stack_trace | text | 堆栈信息 |
| context | jsonb | 错误上下文 |
| diagnosis | jsonb | 诊断结果 |
| resolution | varchar(20) | 处理方式：retry/degrade/absorb/escalate/isolate |
| is_resolved | boolean | 是否已解决 |
| created_at | timestamp | 发生时间 |

### circuit_breakers 表

| 字段 | 类型 | 说明 |
|------|------|------|
| id | bigint | 主键 |
| target_type | varchar(20) | 目标类型：agent/skill |
| target_name | varchar(100) | 目标名称 |
| state | varchar(20) | 状态：closed/open/half_open |
| failure_count | integer | 连续失败计数 |
| success_count | integer | 连续成功计数 |
| last_failure_at | timestamp | 最后失败时间 |
| last_state_change_at | timestamp | 最后状态变更时间 |
| config | jsonb | 熔断配置 |

## 验收标准

1. 错误自动分类为四种类型，分类准确率 ≥ 90%
2. 智能重试根据错误类型选择正确策略
3. 降级执行在 Agent/技能不可用时正确降级
4. 错误传播按配置策略正确处理
5. 自愈诊断生成有效的诊断报告和修复建议
6. 熔断器在连续失败时正确熔断和恢复
7. 补偿事务正确回滚已执行操作的副作用
