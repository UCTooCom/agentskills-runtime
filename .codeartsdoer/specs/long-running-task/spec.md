# 长时间运行任务系统需求规格

## 项目背景

当前 agentskills-runtime 的 Agent 执行是同步的、短生命周期的，无法支持长时间运行的任务（如大规模代码重构、全项目测试、多轮评估-改进循环）。AI 驱动开发框架需要支持执行时间从分钟级到小时级甚至天级的任务，且在执行过程中支持暂停、恢复、检查和干预。

## 核心问题

1. **无持久化执行状态**: Agent 执行状态仅存在于内存，进程重启后丢失
2. **无检查点机制**: 长任务无法保存中间进度，失败后只能从头开始
3. **无进度通知**: 长任务执行过程中用户无法了解进度
4. **无暂停/恢复**: 长任务无法暂停后恢复继续执行
5. **无超时与取消策略**: 缺乏灵活的超时控制和优雅取消机制
6. **无任务队列**: 无法管理待执行的长任务队列和优先级

## 功能需求

### REQ-LRT-001: 任务持久化与恢复

- Agent 执行状态持久化到 agent_tasks 表，包含完整的上下文快照
- 进程重启后可从持久化状态恢复执行
- 支持序列化/反序列化 Agent 的对话历史、工具调用记录、中间结果
- 恢复后 Agent 继续从断点执行，不重复已完成步骤

### REQ-LRT-002: 检查点机制

- 支持自动检查点（每 N 步或每 M 分钟自动保存）
- 支持手动检查点（API 触发保存当前状态）
- 检查点包含：Agent 上下文、已完成步骤列表、中间结果、资源引用
- 检查点存储在独立的 checkpoint 表，支持多个检查点版本
- 失败恢复时自动选择最新有效检查点

### REQ-LRT-003: 实时进度通知

- 通过 WebSocket 推送任务进度事件
- 进度事件类型：step_start、step_complete、step_failed、checkpoint_saved、progress_update
- 进度包含：已完成步骤数/总步骤数、当前步骤描述、预估剩余时间、中间结果预览
- 支持 SSE（Server-Sent Events）作为 WebSocket 的轻量替代

### REQ-LRT-004: 暂停/恢复/取消

- 支持暂停长任务（完成当前步骤后暂停，保存检查点）
- 支持恢复暂停的任务（从检查点加载继续执行）
- 支持优雅取消（完成当前步骤后取消，保留已有结果）
- 支持强制取消（立即终止，标记为 cancelled）
- 暂停/恢复/取消通过 API 和 CLI 均可触发

### REQ-LRT-005: 超时与重试策略

- 支持步骤级超时（每个步骤可设置独立超时时间）
- 支持任务级超时（整个任务的最大执行时间）
- 超时后可配置行为：失败 / 暂停 / 继续并标记
- 支持可配置重试策略（最大次数、退避算法、可重试条件）
- 支持指数退避、固定间隔、自定义退避

### REQ-LRT-006: 任务队列与优先级

- 待执行的长任务进入优先级队列
- 支持优先级：critical > high > normal > low > background
- 支持并发限制（最大同时执行的长任务数）
- 支持资源感知调度（根据任务资源需求分配执行槽）
- 支持任务依赖（任务 B 等待任务 A 完成后执行）

### REQ-LRT-007: 子任务树

- 长任务可分解为子任务树（parent_task_id 建立层级）
- 子任务状态向上传播（所有子任务完成 → 父任务完成）
- 支持子任务部分失败策略（任一失败 / 多数成功 / 全部完成）
- 子任务结果自动聚合到父任务

## 非功能需求

- 单个长任务最大执行时间：24 小时（可配置）
- 检查点保存延迟 ≤ 500ms
- 进度通知延迟 ≤ 1s
- 支持最多 100 个长任务并发执行
- 任务队列支持最多 10000 个待执行任务

## 数据模型

### task_checkpoints 表

| 字段 | 类型 | 说明 |
|------|------|------|
| id | bigint | 主键 |
| task_id | bigint | 关联任务 |
| step_index | integer | 检查点步骤索引 |
| agent_context | jsonb | Agent 上下文快照 |
| completed_steps | jsonb | 已完成步骤列表及结果 |
| intermediate_results | jsonb | 中间结果 |
| resource_refs | jsonb | 资源引用（文件路径、临时数据） |
| is_auto | boolean | 是否自动检查点 |
| created_at | timestamp | 创建时间 |

### task_queue 表

| 字段 | 类型 | 说明 |
|------|------|------|
| id | bigint | 主键 |
| task_id | bigint | 关联任务 |
| priority | integer | 优先级（0-100） |
| resource_requirement | jsonb | 资源需求 |
| depends_on | bigint[] | 依赖的任务 ID 列表 |
| status | varchar(20) | 状态：queued/running/completed |
| enqueued_at | timestamp | 入队时间 |
| started_at | timestamp | 开始时间 |

## 验收标准

1. 长任务执行状态持久化，进程重启后可恢复
2. 自动/手动检查点正确保存和恢复
3. WebSocket 实时推送任务进度
4. 暂停/恢复/取消 API 正确工作
5. 步骤级和任务级超时正确触发
6. 重试策略按配置正确执行
7. 优先级队列正确调度任务执行顺序
8. 子任务树状态正确传播和聚合
