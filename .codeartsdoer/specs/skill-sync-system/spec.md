# 需求规格文档

## 文档信息
- **项目名称**: 技能同步系统 (Skill Sync System)
- **版本**: 1.0
- **创建日期**: 2026-03-27
- **作者**: SDD Agent
- **状态**: 草稿

## 1. 概述

### 1.1 项目背景

当前系统中存在两套技能信息存储机制：
1. **文件系统存储**：`skills/` 目录下的技能定义文件（SKILL.md 等）
2. **数据库存储**：`agent_skills` 数据库表中的技能记录

这种双存储机制导致数据不一致、维护困难、同步成本高等问题。参考项目中已有的国际化系统（locales.json + i18 表）的成功实践，需要设计一套技能同步系统，实现文件系统与数据库的双向一致性同步。

### 1.2 项目范围

**包含内容**：
- 技能信息的双向同步机制设计
- 文件系统变更监听与数据库更新
- 数据库变更监听与文件系统更新
- 冲突检测与解决策略
- 同步状态监控与日志记录
- 手动同步触发接口

**不包含内容**：
- 技能执行引擎的改造
- 技能验证逻辑的修改
- 前端界面的开发
- 技能安装/卸载流程的改造

### 1.3 术语定义

| 术语 | 定义 |
|------|------|
| 技能 (Skill) | 一组可被 AI Agent 调用的功能单元，包含 SKILL.md 定义文件和相关资源 |
| 文件系统技能 | 存储在 `skills/` 目录下的技能定义文件集合 |
| 数据库技能 | 存储在 `agent_skills` 表中的技能记录集合 |
| 双向同步 | 文件系统变更自动更新数据库，数据库变更自动更新文件系统 |
| 同步源 | 触发同步操作的变更来源（文件系统或数据库） |
| 冲突 | 同一技能在文件系统和数据库中存在不一致的关键字段值 |

## 2. 系统需求

### 2.1 功能需求

#### 2.1.1 技能信息读取与解析

##### REQ-PARSE-001: SKILL.md 文件解析
**需求描述**: 系统应能够解析 SKILL.md 文件，提取技能元数据（name、description、version、author 等）。

**验收标准**:
- [ ] 能够正确解析 YAML frontmatter 中的元数据
- [ ] 能够提取 name、description、version、author、keywords、tags 等字段
- [ ] 能够处理缺失字段的情况，使用默认值
- [ ] 解析失败时记录详细错误日志

**优先级**: P0

##### REQ-PARSE-002: 技能目录结构识别
**需求描述**: 系统应能够识别技能目录结构，包括 scripts/、references/、assets/ 等子目录。

**验收标准**:
- [ ] 能够检测 scripts/ 目录是否存在
- [ ] 能够检测 references/ 目录是否存在
- [ ] 能够检测 assets/ 目录是否存在
- [ ] 将检测结果映射到数据库字段 scripts_dir_exists、references_dir_exists、assets_dir_exists

**优先级**: P1

#### 2.1.2 文件系统到数据库同步

##### REQ-F2D-001: 新增技能同步
**需求描述**: 当文件系统中新增技能目录时，系统应自动在数据库中创建对应记录。

**验收标准**:
- [ ] 检测到新技能目录时，自动创建数据库记录
- [ ] 数据库记录的 name 字段与技能目录名一致
- [ ] 从 SKILL.md 解析的元数据正确写入数据库
- [ ] source 字段设置为 "filesystem"
- [ ] source_type 字段设置为 "local"
- [ ] 记录同步日志，标记同步源为 "filesystem"

**优先级**: P0

##### REQ-F2D-002: 技能更新同步
**需求描述**: 当文件系统中的 SKILL.md 文件被修改时，系统应自动更新数据库中的对应记录。

**验收标准**:
- [ ] 检测到 SKILL.md 文件修改时，自动更新数据库记录
- [ ] 只更新发生变化的字段
- [ ] 保留数据库中独有的字段值（如 run_count、success_count 等运行时统计字段）
- [ ] 更新 updated_at 时间戳
- [ ] 记录同步日志，标记更新的字段列表

**优先级**: P0

##### REQ-F2D-003: 技能删除同步
**需求描述**: 当文件系统中的技能目录被删除时，系统应自动标记数据库中的对应记录为已删除（软删除）。

**验收标准**:
- [ ] 检测到技能目录删除时，设置数据库记录的 deleted_at 字段
- [ ] 不执行硬删除，保留历史数据
- [ ] 记录同步日志，标记删除操作

**优先级**: P0

#### 2.1.3 数据库到文件系统同步

##### REQ-D2F-001: 新增技能反向同步
**需求描述**: 当数据库中新增技能记录且 source 为 "database" 时，系统应自动在文件系统中创建对应的技能目录和 SKILL.md 文件。

**验收标准**:
- [ ] 检测到新数据库记录时，创建 skills/{name}/ 目录
- [ ] 生成标准格式的 SKILL.md 文件
- [ ] YAML frontmatter 包含所有元数据字段
- [ ] 创建 scripts/、references/、assets/ 目录（根据对应字段判断）
- [ ] 记录同步日志，标记同步源为 "database"

**优先级**: P0

##### REQ-D2F-002: 技能更新反向同步
**需求描述**: 当数据库中的技能记录被更新时，系统应自动更新文件系统中对应的 SKILL.md 文件。

**验收标准**:
- [ ] 检测到数据库记录更新时，更新 SKILL.md 文件
- [ ] 只更新发生变化的字段
- [ ] 保留文件中独有的内容（如详细说明、使用示例等 Markdown 内容）
- [ ] 记录同步日志，标记更新的字段列表

**优先级**: P0

##### REQ-D2F-003: 技能删除反向同步
**需求描述**: 当数据库中的技能记录被软删除时，系统应自动删除文件系统中对应的技能目录。

**验收标准**:
- [ ] 检测到数据库记录软删除时，删除对应的技能目录
- [ ] 删除前备份技能目录到指定位置
- [ ] 记录同步日志，标记删除操作

**优先级**: P1

#### 2.1.4 冲突检测与解决

##### REQ-CONFLICT-001: 冲突检测
**需求描述**: 当同一技能在文件系统和数据库中同时被修改时，系统应检测到冲突并记录。

**验收标准**:
- [ ] 比较文件系统和数据库的 updated_at 时间戳
- [ ] 检测关键字段的不一致（name、description、version、author）
- [ ] 发现冲突时，记录冲突详情到日志
- [ ] 标记技能状态为 "conflict"

**优先级**: P0

##### REQ-CONFLICT-002: 冲突解决策略
**需求描述**: 系统应提供可配置的冲突解决策略。

**验收标准**:
- [ ] 支持策略：latest_wins（最新修改优先）
- [ ] 支持策略：filesystem_wins（文件系统优先）
- [ ] 支持策略：database_wins（数据库优先）
- [ ] 支持策略：manual（手动解决）
- [ ] 默认策略为 latest_wins
- [ ] 配置文件中可指定策略

**优先级**: P1

#### 2.1.5 同步触发机制

##### REQ-TRIGGER-001: 文件系统监听
**需求描述**: 系统应监听 skills/ 目录的文件系统变更事件。

**验收标准**:
- [ ] 监听文件创建事件
- [ ] 监听文件修改事件
- [ ] 监听文件删除事件
- [ ] 监听目录创建事件
- [ ] 监听目录删除事件
- [ ] 事件触发延迟不超过 1 秒

**优先级**: P0

##### REQ-TRIGGER-002: 数据库监听
**需求描述**: 系统应监听 agent_skills 表的数据库变更事件。

**验收标准**:
- [ ] 监听 INSERT 事件
- [ ] 监听 UPDATE 事件
- [ ] 监听 DELETE 事件
- [ ] 使用 PostgreSQL NOTIFY/LISTEN 机制或轮询机制
- [ ] 事件触发延迟不超过 1 秒

**优先级**: P0

##### REQ-TRIGGER-003: 手动同步接口
**需求描述**: 系统应提供手动触发同步的 API 接口。

**验收标准**:
- [ ] 提供 POST /api/v1/uctoo/skill-sync/sync 接口
- [ ] 支持指定同步方向（filesystem-to-database、database-to-filesystem、bidirectional）
- [ ] 支持指定同步范围（all、specific skills）
- [ ] 支持强制同步（忽略冲突检测）
- [ ] 返回同步结果统计

**优先级**: P1

#### 2.1.6 同步状态管理

##### REQ-STATUS-001: 同步状态记录
**需求描述**: 系统应记录每次同步操作的详细状态。

**验收标准**:
- [ ] 记录同步开始时间
- [ ] 记录同步结束时间
- [ ] 记录同步源（filesystem/database）
- [ ] 记录同步方向（f2d/d2f）
- [ ] 记录受影响的技能列表
- [ ] 记录同步结果（success/failure/partial）
- [ ] 记录错误信息（如有）

**优先级**: P1

##### REQ-STATUS-002: 同步日志查询
**需求描述**: 系统应提供同步日志查询接口。

**验收标准**:
- [ ] 提供 GET /api/v1/uctoo/skill-sync/logs 接口
- [ ] 支持按时间范围过滤
- [ ] 支持按同步源过滤
- [ ] 支持按结果状态过滤
- [ ] 支持分页查询

**优先级**: P2

### 2.2 非功能需求

#### 2.2.1 性能需求
- 同步操作响应时间不超过 2 秒（单个技能）
- 批量同步（100 个技能）不超过 30 秒
- 文件系统监听事件处理延迟不超过 1 秒
- 数据库监听事件处理延迟不超过 1 秒

#### 2.2.2 可靠性需求
- 同步失败时自动重试，最多重试 3 次
- 同步操作支持事务，失败时回滚
- 系统重启后能够恢复未完成的同步操作

#### 2.2.3 可维护性需求
- 同步逻辑模块化，便于扩展
- 提供详细的同步日志，便于问题排查
- 支持同步操作的监控和告警

#### 2.2.4 兼容性需求
- 兼容现有的技能目录结构
- 兼容现有的 agent_skills 表结构
- 不影响现有的技能执行流程

### 2.3 约束性需求
- 使用 Cangjie 语言实现
- 使用 PostgreSQL 作为数据库
- 使用现有的 ORM 框架（Fountain ORM）
- 遵循现有的 API 规范（/api/v1/uctoo/...）

## 3. 接口需求

### 3.1 用户接口
无直接用户界面，通过 API 接口交互。

### 3.2 系统接口

#### 3.2.1 同步管理 API
- POST /api/v1/uctoo/skill-sync/sync - 触发同步
- GET /api/v1/uctoo/skill-sync/status - 查询同步状态
- GET /api/v1/uctoo/skill-sync/logs - 查询同步日志
- POST /api/v1/uctoo/skill-sync/resolve-conflict - 解决冲突

#### 3.2.2 内部事件接口
- 文件系统事件 → 同步管理器
- 数据库事件 → 同步管理器
- 同步管理器 → 文件系统操作
- 同步管理器 → 数据库操作

### 3.3 数据接口
- 读取 skills/ 目录下的 SKILL.md 文件
- 读写 agent_skills 数据库表
- 写入 skill_sync_logs 日志表（新增）

## 4. 数据需求

### 4.1 数据模型

#### 4.1.1 技能同步日志 (skill_sync_logs)
| 字段 | 类型 | 说明 |
|------|------|------|
| id | UUID | 主键 |
| skill_id | UUID | 关联的技能 ID |
| skill_name | String | 技能名称 |
| sync_source | String | 同步源（filesystem/database） |
| sync_direction | String | 同步方向（f2d/d2f） |
| sync_status | String | 同步状态（running/success/failure/partial） |
| affected_fields | JSON | 受影响的字段列表 |
| error_message | Text | 错误信息 |
| started_at | DateTime | 开始时间 |
| finished_at | DateTime | 结束时间 |
| created_at | DateTime | 创建时间 |

#### 4.1.2 同步配置 (skill_sync_config)
| 字段 | 类型 | 说明 |
|------|------|------|
| id | UUID | 主键 |
| config_key | String | 配置键 |
| config_value | String | 配置值 |
| description | Text | 配置说明 |
| created_at | DateTime | 创建时间 |
| updated_at | DateTime | 更新时间 |

### 4.2 数据存储
- 新增 skill_sync_logs 表存储同步日志
- 新增 skill_sync_config 表存储同步配置
- agent_skills 表新增 source 字段标记数据来源
- agent_skills 表新增 sync_status 字段标记同步状态

### 4.3 数据迁移
- 为现有 agent_skills 记录设置 source = 'filesystem'
- 为现有 agent_skills 记录设置 sync_status = 'synced'
- 初始化 skill_sync_config 表的默认配置
