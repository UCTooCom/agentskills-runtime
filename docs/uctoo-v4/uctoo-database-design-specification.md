# DB版本 v4.0.0 Beta  

1. 表名、列名均采用‌小写命名法，例如，created_at、updated_at、deleted_at。单词间以下划线连接。应该避免使用编程语言的关键字和保留字。  
2. 所有数据库表均包含created_at、updated_at、deleted_at、creator四个列，代表创建时间、更新时间、软删除时间、创建人ID，creator列关联uctoo_user表id列。  
3. 数据库表的字段备注信息，可用于生成国际化的翻译资源文件数据，备注中包含中文句号"。"的，取句号前内容用于多语言资源文件，没有句号的取全部备注用于多语言资源文件。字段无备注信息则用字段名    

## 支持多种数据库  
uctoo通过ORM层适配支持多种类型的数据库，默认采用postgresql。框架自带的uctoo数据库是框架正常运行依赖的框架数据库，框架正常运行需先在.env中配置uctoo数据库的连接信息。如需集成多数据库可从管理后台->数据库连接菜单，添加更多业务数据库连接信息,更多业务数据库连接信息保存在db_connection表。


## 统一数据库信息
不同数据库对数据库信息的定义并不一致，uctoo采用了统一数据库信息格式的方案，为低代码/无代码/可视化代码生成提供一致的数据源。db_info表保存框架所连接的数据库的自完备信息。db_info表的设计与默认数据库类型postgresql对数据库信息描述的设计一致。其他类型数据库，可转换对应的数据库信息对应至db_info表保存。通过命令行工具loadDbInfo可将当前连接的数据库信息加载至db_info表。数据库连接信息优先使用db_connection表中保存的数据，如db_connection表中没有与数据库名参数一致的数据库连接信息，则采用.env文件配置的数据库连接信息，请确保数据库连接信息配置正确。todo:目前已实现postgresql和mysql数据库信息加载，其他类型数据库开发者可自行扩展/api/v1/uctoo/db_info/load-db-info API的方法。
